IEEE Attribute Descriptors
  • Traceable: The origin of each requirement is clear
  • Feasible: Requirements can be implemented by the team
  • Singular: Each requirement specifies only one need/ function to be implemented
  • Verifiable: Can check the software meets the requirements
  • Consistent: The requirements should agree with the product brief
  • Unambiguous: Every requirement has only one interpretation
  • Complete: The requirements fully describe the products functionality
  • Correct: Every requirement stated is one the software shall meet
  • Affordable: Each requirement can be completed within our monetary budget and time available
